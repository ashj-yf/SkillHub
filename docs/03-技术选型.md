## 二、后端技术选型

### 2.1 Rust + Axum（推荐）

**优势：**
- 高性能：零成本抽象，无 GC 停顿
- 类型安全：编译时错误检测
- 单二进制部署：无需运行时环境
- CLI 复用：后端代码可直接用于 CLI

**劣势：**
- 学习曲线较陡
- 生态相对年轻

### 2.2 Go + Gin（备选）

**优势：**
- 开发效率高
- 生态成熟
- 部署简单

**劣势：**
- 错误处理繁琐
- 泛型支持有限

### 2.3 推荐结论

**Rust + Axum**，理由：
1. CLI 和后端可共享代码
2. 性能优势明显
3. 类型安全减少运行时错误
