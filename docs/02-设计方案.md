## 三、各层详细设计

### 3.1 感知层 (Context Layer)

**职责：** 实时感知用户上下文，为智能推荐提供输入

**组件：**
| 组件 | 功能 | 实现方式 |
|------|------|----------|
| 项目检测器 | 分析项目技术栈 | 文件扫描 + heuristics |
| 意图识别器 | 检测用户当前任务 | 代码上下文 + 行为模式 |
| 上下文聚合器 | 整合多源信息 | 实时数据流处理 |

### 3.2 智能层 (Intelligence Layer)

**职责：** 提供智能推荐、语义搜索、技能关联能力

**组件：**
| 组件 | 功能 | 实现方式 |
|------|------|----------|
| 技能图谱 | 技能间的继承/组合关系 | 图数据库 (Neo4j / 内置) |
| 向量引擎 | 语义搜索 | Qdrant / Milvus |
| 推荐引擎 | 个性化推荐 | 协同过滤 + 内容推荐 |

### 3.3 存储层 (Storage Layer)

**职责：** 持久化存储技能内容和元数据

**组件：**
| 组件 | 功能 | 实现方式 |
|------|------|----------|
| Git 仓库 | 存储技能文件 | Gitea / GitLab CE |
| 向量数据库 | 存储语义索引 | Qdrant / Milvus |
| 关系数据库 | 存储元数据/权限 | PostgreSQL |

### 3.4 分发层 (Delivery Layer)

**职责：** 向用户提供多渠道的技能分发

**组件：**
| 组件 | 功能 | 用户场景 |
|------|------|----------|
| CLI 工具 | 命令行操作 | 开发者日常使用 |
| Web 市场 | 浏览/搜索/安装 | 非技术人员使用 |
| IDE 插件 | 实时推荐 | 编码时辅助 |
| API/SDK | 系统集成 | 第三方工具接入 |

## 四、核心创新点

### 4.1 技能图谱（Skill Graph）

技能可以 **继承** 和 **组合**，形成知识网络：

```
                    ┌─────────────────┐
                    │   代码审查      │
                    │  (root skill)   │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
      ┌───────────────┐ ┌───────────────┐ ┌───────────────┐
      │ Python 代码审查│ │ 前端代码审查  │ │ 安全代码审查  │
      │  (extends)    │ │  (extends)    │ │  (extends)    │
      └───────┬───────┘ └───────────────┘ └───────────────┘
              │
              ▼
      ┌───────────────┐
      │ Django 代码审查│
      │  (extends)    │
      └───────────────┘
```

### 4.2 上下文感知（Context Sensing）

CLI 自动感知当前项目，智能推荐相关技能：

```bash
$ cd my-django-project
$ skillhub sense

🔍 检测到：Python + Django + PostgreSQL 项目

💡 推荐技能：
   • django-best-practices  (相关度: 94%)
   • python-security        (相关度: 87%)
   • postgresql-optimization (相关度: 82%)

⚡ 一键应用：skillhub apply django-best-practices
```

### 4.3 多格式兼容（Multi-Format）

一个技能，多端适配 —— 写一次，各 AI 工具都能用：

```
skillhub/
├── skills/
│   ├── code-review/
│   │   ├── skill.yaml          # 统一元数据
│   │   ├── claude-skill.md      # Claude Code 格式
│   │   ├── cursor-rule.md       # Cursor 格式
│   │   └── copilot-instruct.md  # Copilot 格式
│   └── ...
```

### 4.4 智能演化（Evolution）

技能可记录学习来源，持续优化：

```yaml
# skill.yaml
name: python-security
version: 2.1.0
extends: security-base        # 继承基础技能
composes:                     # 组合其他技能
  - bandit-rules
  - owasp-top10
learned_from:                 # 记录来源（用于推荐）
  - incidents: [SEC-123, SEC-456]
  - best_practices: [BP-001]
```

## 五、用户旅程

### 场景 1：新项目初始化
```bash
$ cd new-project
$ skillhub init --detect

🔍 检测到项目类型：React + TypeScript + Vite

📦 推荐初始化技能包：
   ✓ frontend-best-practices
   ✓ react-patterns
   ✓ typescript-strict

? 应用这些技能？(Y/n)
```

### 场景 2：Web 浏览安装
1. 访问 Web 市场
2. 搜索 "React 性能优化"
3. 查看技能详情、评分、使用案例
4. 点击"安装" → 自动同步到本地项目

### 场景 3：实时推荐
1. 开发者在 VS Code 中编写代码
2. IDE 插件检测到可能的性能问题
3. 弹出提示："检测到潜在性能问题，推荐安装 react-performance 技能"
4. 一键接受 → 技能自动加载
