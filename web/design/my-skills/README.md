# 我的技能 - UI 设计文档

## 一、用户场景

### 目标用户
- 技能开发者：管理自己发布的技能

### 用户目标
- 查看自己创建的技能
- 编辑技能信息
- 发布新版本
- 删除技能

### 使用场景
- 查看自己发布的所有技能
- 更新技能描述或标签
- 发布技能新版本
- 下架不再维护的技能

## 二、用户旅程图

```mermaid
graph TD
    A[访问我的技能] --> B[加载技能列表]
    B --> C{加载结果}
    C -->|失败| D[显示错误]
    C -->|成功| E{是否有技能}

    E -->|无| F[显示空状态]
    F --> G[引导创建技能]

    E -->|有| H[显示技能列表]
    H --> I{用户操作}

    I -->|编辑| J[打开编辑表单]
    J --> K[保存修改]
    K --> L[更新列表]

    I -->|发布版本| M[打开版本上传]
    M --> N[上传文件]
    N --> O[发布成功]

    I -->|删除| P[确认删除]
    P --> Q{确认}
    Q -->|是| R[删除技能]
    Q -->|否| H
```

## 四、状态设计

### 4.1 加载状态
- 显示 loading 动画
- 禁用创建按钮

### 4.2 空数据状态
- 显示空状态插图
- 显示引导文案
- 提供创建按钮

### 4.3 错误状态
- 显示错误提示
- 提供重试按钮

### 4.4 成功状态
- 操作成功后显示 Toast 提示
- 自动刷新列表

## 五、API 依赖

| API | 用途 | 状态 |
|-----|------|------|
| GET /api/users/me/skills | 获取我的技能列表 | ✅ 已实现 |
| PUT /api/skills/{slug} | 更新技能 | ✅ 已实现 |
| DELETE /api/skills/{slug} | 删除技能 | ✅ 已实现 |
| POST /api/skills/{slug}/versions | 创建新版本 | ✅ 已实现 |
